2. Diskussion der Feature-Liste

* Autonome Aktionen (mit Infos von Frontend, aber auch manuelle Initiierung durch Frontend möglich)
        * Update, Upgrade       -> no goal
        * Einrichtung
        * Reboot der VM / des PCs
        * Suspend / Resume der VM
        * Versionskontrolle (Änderungen in VM-Image ans Frontend übertragen)        -> no goal
* Aktionen (Frontend -> Backend)
        * Load / Unload Image
        * One-Time-Passwörter für z.B. SSH, Key-Management
* Aktionen (Backend -> Frontend)
        * Bei abgeschlossener Einrichtung / Start: Lebenszeichen an Frontend, dass Maschine bereit
        * Heartbeat
        * Failover, Master/Slave        -> no goal
        * Systemfehler, Logs
* Statusabfragen
        * Ist Image geladen?
        * Welches Image ist geladen?
        * ComputerID zur Verknüpfung Seller, Buyer, Transaktion im Frontend
        * Effektive Anfangs- / Endzeiten (Wann startete Benutzung)für Abrechnung im Frontend
        * Systemstatus
                * Synchronität  -> no goal
        * System-Stats (CPU, Auslastung, ...) zur Klassifikation der PCs im Frontend
